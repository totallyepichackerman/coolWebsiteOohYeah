ATCS FINAL PROJECT:

Goals

I hope to create a top-down shooter game where the character is the center of the screen and a large map is what moves to reveal more. I have the following major milestones that I would like to reach:
1. WASD to move, the character will always face the mouse
2. Have different weapons that can be selected like an inventory
3. Enemies that can fight back when they see the character
4. An hp and healing system for when the enemies injure the player

Plan

A large issue I will face is a week long school trip to Taiwan with my chinese class. This will make it extremely difficult to complete the project in time because I will miss a significant amount of time I have to work on it. I hope to complete the map and movement before I leave, so that I can focus on the more game specific stuff once I come back.

Journal

Week One: I created the character in the middle of the screen, not using any objects yet. I also got the rotation of the not weapon around the character mostly working, which took a lot longer than I was expecting. It still has issues when pointing straight up and straight down.
Week Two: I was in Taiwan, and was extremely busy. I was able to fix the bugs in the rotation of the not weapon but that was it
Week Three: I brought back the tilemap class from last year's final project and made all of the necessary adjustments to make it work with the rest of the project. I also created basic colored square graphics for now. I then made all of the classes necessary for the details of the game, which was the player, notweapon, and enemy classes.
Week Four: This is the week of the presentation, meaning I only had Monday and Tuesday. In that time I began working on integrating the notweapon class, which included the introduction of the notbullet class. This had similar rotation issues as rotating the notweapon itself did previously, which took a lot of time. I spent Tuesday afternoon trying a bunch of different ideas to try and fix the issue so that I could at least show off the notweapon firing. I ended up being able to create an array of three different weapons, which can be selected with the 1, 2, and 3 keys.

Summary

By the end of the project I completed the first two milestones, and had made progress towards the other two. There is currently a class for enemies and the character and enemies both have hp variables, although I didn't have enough time to implement them. As I predicted in the project planning, the trip to Taiwan messed up my schedule and caused me to run out of time. I also underestimated how difficult the trig would be when making the weapons, which took up a lot of the time that I did have. If I were to continue the project, I would definitely implement the enemies and healing mechanics. I would also update the graphics, and use icons to make it more clear that there are different weapons.
